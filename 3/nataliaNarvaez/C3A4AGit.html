GIT

Permite hacer backup de archivos y compartir con los colaboradores

¿Que es Git? Software de control de versiones pensado para la eficiencia y confiabilidad del mantenimiento de aplicaciones cuando estas tienen un gran número de archivos de código fuente. Lleva registro de los cambios de los archivos y coordina el trabajo que se hace sobre ellos.

Historial completo de las versiones de un mismo archivo sin necesidad de crear varias copias. y se pueden compartir los archivos y seguimiento de quien hace modificaciones.  Es similar a Google Drive pero no es lo mismo porque es para código.

Comandos de git

$ git init : comando para iniciar un repositorio vacío, genera un repositorio local en la carpeta que tengo los archivos. Un repositorio es un almacén de archivos donde se irán almacenando tus archivos en pequeños paquetes que se llaman Commits donde se hará seguimiento de los historiales de cambios que se hicieron en el proyecto. y por eso necesito agregar mi identidad al repositorio.

$ git config user.name “ailahtan” : para crear tu nombre de usuario, pero primero debe estar creado el repositorio.

$ git config user.mail “miCorreo@mail.com” : por ultimo agrego mi correo para que mi repositorio en git me reconozca.

Si después de ingresar mis datos de usuario en el repositorio agrego a la linea de codigo “ - - “ Lo que pasará es que me tomara como un usuario global para cualquier repositorio que cree:  

$ git config --global user.email "ailahtan999@gmail.com"

ls -a  : Me muestra los archivos ocultos de la carpeta

git remote add origin https://github.com/DH/RepoRemoto // apunta al repositorio remoto

¿Cómo agregar archivos al repositorio?

git add : comando para agregar archivos a mi repositorio de git

git status : te dice el estado de los archivos respecto al repositorio

git add .  : comando para agregar todos los archivos a git que contiene esa carpeta

Cuando se modifica un archivo este es una “nueva versión” por lo que este nuevo archivo no tendrá seguimiento. Ese es el control de versiones, como tiene un cambio para poder agregarlo de nuevo hay que ejecutar el git add . para que sea actualizado.

Creación de “commits”

Generar una marca, un punto cronológico del proyecto que nos permita volver si lo necesitamos a una versión anterior.

$ git commit -m “mensajeEntreComillas” La idea es con el mensaje, describir los cambios que se hicieron hasta ese momento.

En un commit sólo entran los archivos a los que les he hecho previamente add.

git log muestra el historial de todos los commits

git push origin master comando para enviar mis archivos del repositorio local al repositorio remoto, cuando ejecuto este bloque de código me pedirá usuario y contraseña que a pesar de que lo estaré tipeando, no aparecerá

ramas  una versión 2 de tu proyecto donde puedes probar cosas nuevas

Bajando archivos de github

git clone: te permite crear una copia exacta en tu computadora de todos los archivos existentes pero necesita la url completa de en donde se encuentran guardados los archivos, ejemplo:

git clone https://github.com/ailahtan/ctd.git

Después, git descargara la copia de los archivos,  esto solo se hace cuando voy a bajar los archivos por primera vez, pero si quiero actualizar los archivos que ya bajé, el comando será otro.

git remote -v : comando que me mostrará con que link de repositorio remoto estoy conectado.

SINCRONIZAR O ACTUALIZAR AVANCES

git pull : comando para actualizar archivos de un git clone ejecutado previamente, el cual no bajará todo nuevamente sino que actualizará los archivos que hayan sido modificados previamente,  y completo con origin master  para indicar que quiero llamar todos los archivos nuevos y con cambios, entonces:

$ git pull origin master
